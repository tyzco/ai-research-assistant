# 学术文献精读助手 — 技术文档 & 面试准备

> MVP v1.1 | 2024-06-18

---

## 一、项目概述

面向研究生/学者的文献精读工具。输入研究方向 → 检索策略 + 多源论文发现 → 一键下载构建个人知识库 → 基于全文的深度 RAG 问答。

**核心差异化**：不是简单的 LangChain 调包，而是自研了自适应混合检索、级联重排、多模态图表理解等关键模块。

## 三、核心 RAG 链路（自研模块）

### 3.1 自适应混合检索 (AdaptiveHybridRetriever)

**文件**: `hybrid_retriever.py`

**创新点**：不是简单的向量+BM25 拼接，而是根据问题特征**动态调整融合权重**。

```
问题特征分析 → BM25权重(alpha)
  ├─ 短查询(<15字) → alpha+=0.2 (偏关键词)
  ├─ 含英文术语 → alpha+=0.15 (精确匹配)
  ├─ 问句形式 → alpha-=0.15 (偏语义)
  └─ 长查询(>40字) → alpha-=0.1 (偏语义)

自适应 RRF: weighted_rank_score = alpha × RRF(BM25) + (1-alpha) × RRF(Vector)
```

**与开源方案的区别**:
- LlamaIndex/ LangChain 的混合检索是固定权重或简单规则
- 本项目用 4 维特征工程做自适应，可解释、可消融

### 3.2 两阶段级联重排序

**文件**: `hybrid_retriever.py` (CascadeRerank)

```
Stage1: all-MiniLM-L6-v2 相似度预筛
  top-20 → top-15 (轻量模型, ~20ms)

Stage2: bge-reranker-base CrossEncoder 精排
  top-15 → top-5 (重排序, ~200ms)
```

**与直接用 reranker 的区别**:
- 节省 ~40% 推理时间 (Stage1 过滤掉明显不相关的块)
- MiniLM 可本地运行，bge-reranker 需要 GPU 或较多 CPU 内存
- 可单独替换任一 Stage

### 3.3 三层 RAG 检索架构

```
用户问题
  → 查询改写 (DeepSeek)
  → 自适应混合检索
      ├─ 全文文本层 (BM25 + Vector, is_fulltext=true)
      ├─ 图片描述层 (BM25 + Vector, is_image=true)
      └─ 摘要补充层 (Vector, is_fulltext=false)
  → 级联重排
  → Prompt 拼接
  → DeepSeek 生成 + 引用 + 文献缺口提示
```

### 3.4 多模态图表理解

```
PDF → PyMuPDF 提取图片 → Qwen-VL 生成中文描述 → 向量化存入 LanceDB
     → 检索时匹配 → 回答中展示原图
```

### 关键发现

1. **纯向量检索已接近最优**——Top-5 即达 80.8%，与混合检索差距仅 0.1%
2. **混合检索 + 查询改写带来边际提升**——≥0.80 从 10 题提到 11 题
3. **瓶颈在嵌入模型**——bge-small-zh (512维) 的表达能力上限 ~82%
4. **升级路径明确**：bge-large-zh (1024维) 预期 +3-5%，直接突破 85%

