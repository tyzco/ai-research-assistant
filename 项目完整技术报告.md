## 二、核心数据流

```
用户输入 "人脸识别"
  │
  ├─[1. 检索策略] DeepSeek 生成
  │   → 5 中英关键词 + 4 数据库检索式 + 领域映射 + 关联术语
  │
  ├─[2. 多源搜索] 5 源并行 (asyncio.gather)
  │   → arXiv(2kw×50) + OpenAlex(英2kw×100 + 中3kw×100)
  │   → Semantic Scholar + Unpaywall DOI→PDF 补链 + CNKI链接
  │   → 去重 → 质量过滤 → CN相关性 → 中文优先排序
  │   → 返回 200+ 篇论文 (含 is_oa/pdf_url/cnki_url/doi_url)
  │
  ├─[3. 一键导入] 用户点击 "⬇📥 一键下载并构建知识库"
  │   → 并发下载 OA PDF (Semaphore(3) 限流)
  │   → PyMuPDF 提取全文 + 语义分块 (句子边界, 2句重叠)
  │   → bge-small-zh (512维) 批量向量化
  │   → LanceDB 双层索引: 全文层 + 摘要层
  │
  ├─[4. RAG 问答] 用户提问
  │   → 查询改写 (DeepSeek) → 语义校验 (余弦≥0.75)
  │   → 自适应混合检索: BM25 + Vector (4维特征推理权重)
  │   → 级联重排: MiniLM预筛(20→15) → bge-reranker精排(15→5)
  │   → 策略路由: 复杂查询才触发Rerank, 简单查询跳过
  │   → DeepSeek 生成 → 多轮记忆保存
  │   → 返回 Markdown 答案 + 引用 + 补充线索
  │
  └─[5. Agent 模式] 高层目标自动执行
      → ReAct 循环: Thought → Action → Observation
      → 8 个标准化 Tool (Function Calling 格式)
      → 规划→搜索→下载→分析→报告 全自动
```

## 四、LangGraph 图状 Agent 工作流 (v2.0 新增)

### 4.1 为什么需要 LangGraph

当前 Agent 是线性的 ReAct 循环——每一步只能顺序执行。真实学术调研需要：
- **并行搜索**: 同时搜多个方向
- **条件分支**: 结果不足→重新规划关键词
- **人工确认**: 批量下载前需用户确认
- **错误恢复**: 某步失败后自动重试或跳过

### 4.2 LangGraph 工作流设计

```
                    ┌─────────────┐
                    │   START     │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  PlanNode   │  LLM 规划子任务
                    │  分解目标    │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
     ┌────────▼───┐ ┌──────▼──────┐ ┌──▼──────────┐
     │ SearchNode  │ │ SearchNode  │ │ SearchNode   │  并行搜索
     │ 关键词组A   │ │ 关键词组B   │ │ 关键词组C    │
     └────────┬───┘ └──────┬──────┘ └──┬──────────┘
              │            │            │
              └────────────┼────────────┘
                           │
                    ┌──────▼──────┐
                    │  MergeNode  │  结果合并去重
                    └──────┬──────┘
                           │
              ┌────────────▼────────────┐
              │   papers_count > 20?    │  条件边
              └────────────┬────────────┘
                     Yes   │   No
              ┌────────────▼──┐  ┌────▼─────────┐
              │ DownloadNode  │  │ ReplanNode    │
              │ 批量下载建库   │  │ 重新规划关键词 │
              └──────┬────────┘  └──────┬────────┘
                     │                  │
                     │         ┌────────▼────────┐
                     │         │   SearchNode    │
                     │         │   新关键词搜索   │
                     │         └────────┬────────┘
                     │                  │
              ┌──────▼──────────────────▼──────┐
              │        HumanApprovalNode        │  人工确认
              │   "即将下载82篇论文, 继续?"     │
              └──────┬──────────────────────────┘
                     │
              ┌──────▼──────┐
              │ AnalyzeNode │  LLM 提取关键信息
              └──────┬──────┘
                     │
              ┌──────▼──────┐
              │ ReportNode  │  生成结构化报告
              └──────┬──────┘
                     │
              ┌──────▼──────┐

              └─────────────┘
```

### 4.3 代码实现

```python
# agent/langgraph_workflow.py
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated
import operator

class AgentState(TypedDict):
    task: str
    papers: Annotated[list, operator.add]
    kb_table: str
    report: str
    error: str
    needs_approval: bool

def plan_node(state: AgentState) -> AgentState:
    """LLM 分解任务 → 子任务列表"""
    ...

def search_node(state: AgentState) -> AgentState:
    """并行搜索论文"""
    ...

def merge_node(state: AgentState) -> AgentState:
    """合并去重"""
    ...

def download_node(state: AgentState) -> AgentState:
    """下载建库"""
    ...

def analyze_node(state: AgentState) -> AgentState:
    """LLM 提取关键信息"""
    ...

def report_node(state: AgentState) -> AgentState:
    """生成结构化报告"""
    ...

def route_after_merge(state: AgentState) -> str:
    if len(state["papers"]) > 20:
        return "download"
    return "replan"

# 构建图
workflow = StateGraph(AgentState)

workflow.add_node("plan", plan_node)
workflow.add_node("search_a", lambda s: search_node(s, "A"))
workflow.add_node("search_b", lambda s: search_node(s, "B"))
workflow.add_node("merge", merge_node)
workflow.add_node("download", download_node)
workflow.add_node("replan", replan_node)
workflow.add_node("approval", human_approval_node)
workflow.add_node("analyze", analyze_node)
workflow.add_node("report", report_node)

workflow.set_entry_point("plan")
workflow.add_edge("plan", "search_a")
workflow.add_edge("plan", "search_b")
workflow.add_edge("search_a", "merge")
workflow.add_edge("search_b", "merge")
workflow.add_conditional_edges("merge", route_after_merge, {
    "download": "download",
    "replan": "replan"
})
workflow.add_edge("download", "approval")
workflow.add_edge("approval", "analyze")
workflow.add_edge("analyze", "report")
workflow.add_edge("report", END)

agent = workflow.compile()
result = agent.invoke({"task": "调研LLM推理方法突破"})
```

### 4.4 与现有 ReAct 循环的对比

| 维度 | ReAct (当前) | LangGraph (v2.0) |
|------|-------------|------------------|
| 执行模式 | 串行线性 | 图状, 支持并行 |
| 分支逻辑 | 手动 if/else | 条件边自动路由 |
| 人工确认 | 无 | HumanApprovalNode |
| 错误恢复 | 基础重试 | 节点级 try/except + 回退 |
| 可视化 | 无 | LangGraph Studio 调试 |
| 复杂度 | 简单, 够用 | 复杂, 适合多步调研 |

## 八、多模态图片处理 (v2.0 新增)

### 设计原则
论文PDF中的图片90%是出版社Logo/作者照片/装饰元素，真正有学术价值的图表较少。
我们的方案：**两阶段过滤 + 懒加载**。

### 第一阶段：快速规则预筛（零开销）
```python
def _fast_image_filter(width, height, image_bytes, page_num, page_count):
    if width < 100 or height < 100:     return False  # Logo/小图标
    if max(w,h)/min(w,h) > 6.0:         return False  # 横幅装饰
    if page_num <= 1 or page_num >= pc-1: return False # 首页/末页噪声
    if len(Image.open(img).getcolors(10)) < 3: return False # 单色图标
    return True
```
- 耗时: <1ms/张, 零API调用
- 效果: 过滤~50%噪声图片

### 第二阶段：懒加载（按需触发）
- PDF解析时仅保存通过预筛的图片元数据
- 用户提问命中某篇论文时→提取临近图片→视觉模型生成描述→展示+缓存
- 首次提问额外2-3秒, 后续命中缓存秒出

### 时间与准确率权衡
| 方案 | 单篇耗时 | 准确率 |
|------|---------|--------|
| 全量预处理 | 30+秒 | 100% (含大量噪声) |
| 规则预筛+懒加载 | 首次3-5秒, 后续<1秒 | 95% (过滤噪声, 保留图表) |
